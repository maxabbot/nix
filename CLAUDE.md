# CLAUDE.md

## Project Overview

NixOS workstation configuration using **Nix Flakes** and **Home Manager**.
Declarative, reproducible, and fully idempotent.

## Repository Structure

```
flake.nix                    # Entry point — inputs, outputs, nixosConfigurations
setup.sh                     # NixOS rebuild wrapper (auto-detects hostname)
hosts/
  home-desktop/              # Gaming workstation (RTX 40-series, Hyprland)
    default.nix              #   Host config — imports modules, sets options
    hardware-configuration.nix  #   Generated by nixos-generate-config
  work-laptop/               # Dev laptop (Sway, TLP)
  minimal/                   # Headless / base only
modules/
  nixos/                     # System-level NixOS modules (replaces Ansible roles)
    base.nix                 #   Core packages, networking, fonts, services
    development.nix          #   Languages, containers, cloud CLIs, DB tools
    productivity.nix         #   DE (Hyprland/Sway), audio, apps, browsers
    nvidia.nix               #   Drivers, CUDA, kernel params, DRM modesetting
    gaming.nix               #   Steam, Wine, Proton, controllers, performance
  home/                      # Home Manager modules (replaces Chezmoi)
    default.nix              #   Imports all HM sub-modules
    shell.nix                #   Zsh, Starship, aliases, tmux
    editor.nix               #   Zed (primary), VSCode
    apps.nix                 #   Kitty, btop, mpv, Zathura, Yazi
    theme.nix                #   Gruvbox Material — GTK, Qt/Kvantum, cursor
    wm/
      hyprland.nix           #   Hyprland config (parameterised by monitors)
      sway.nix               #   Sway config
home/
  max/default.nix            # Home Manager user config (git, gammastep, direnv, etc.)
overlays/default.nix         # Custom package overrides
pkgs/default.nix             # Local derivations not in nixpkgs
bootstrap/                   # Legacy archinstall configs (kept for reference)
system/                      # Legacy Ansible roles (kept for reference)
user/                        # Legacy Chezmoi dotfiles (kept for reference)
docs/                        # Post-installation guide, reference checklists
```

## Key Conventions

### NixOS Modules

- Each module exposes options under `custom.<name>.*`.
- All options are **off by default** — hosts explicitly enable what they need.
- Host configs live in `hosts/<name>/default.nix` and import the relevant modules.
- `hardware-configuration.nix` is a placeholder — replace with the output of
  `sudo nixos-generate-config --root /mnt` on the target machine.

### Module option hierarchy

```nix
custom.base.{enable, username, timezone, powerManagement, btrfsSnapshots, firewall}
custom.development.{enable, containers.podman, containers.libvirt, database.*, cloudTools}
custom.productivity.{enable, compositor, creativeApps, streamingTools, communicationApps}
custom.nvidia.{enable, open, cuda}
custom.gaming.{enable, wineExtras, streaming, apollo}
```

### Home Manager

- HM is wired as a NixOS module (`home-manager.nixosModules.home-manager`).
- `home/max/default.nix` receives `specialArgs`: `machineType`, `compositor`,
  `monitors`, `git`, `location`, `inputs`.
- Monitor parameterisation via `custom.hm.monitors.{primary, secondary}` options
  (defined in `modules/home/wm/hyprland.nix`).

### Running a rebuild

```bash
sudo nixos-rebuild switch --flake /etc/nixos#home-desktop
sudo nixos-rebuild switch --flake /etc/nixos#work-laptop
sudo nixos-rebuild switch --flake /etc/nixos#minimal
```

### Shell Scripts

- Linted with **shellcheck** (CI enforced). Config: `.shellcheckrc` enables `external-sources=true`.
- Use `bash` with proper quoting and error handling.

## CI/CD

GitHub Actions (`.github/workflows/ci.yml`):
- `nix flake check` — evaluates all NixOS configurations for correctness
- `nixfmt-rfc-style` — Nix formatting check
- `shellcheck` — lints all `.sh` files

## Adding Packages

1. **System package** → add to `environment.systemPackages` in the relevant `modules/nixos/<role>.nix`.
2. **User package** → add to `home.packages` in `home/max/default.nix`.
3. **Optional package** → gate behind a new `mkEnableOption` in the module options, then enable in the host config.
4. **Not in nixpkgs** → add a custom derivation to `pkgs/default.nix` and expose via `overlays/default.nix`.

## Adding a New Module

1. Create `modules/nixos/<name>.nix` with `options.custom.<name>.*` and `config = lib.mkIf cfg.enable { ... }`.
2. Import the module in the relevant host's `default.nix`.
3. Set `custom.<name>.enable = true;` and any other options in the host config.

## Security

- Never commit credentials — `user-credentials.json` and `github_pat` are in `.gitignore`.
- `result` and `result-*` (Nix build outputs) are gitignored.
- No hardcoded secrets anywhere in the repo.
